# D3 Artifact Contract

An authoritative D3 result requires a preregistration frozen before tensor
loads, an exact-path guard, immutable prompt-A/B cached features, exact L0
decoder states, the P0 targets, square mapping, hard assignment, complete D1R
reference tensors, a passing one-step trace, synchronous trajectory artifacts,
and fresh-process checkpoint replay.

Thayer-D3R satisfies only the pre-run portions of that contract. It preserves
the preregistration, environment and campaign snapshots, exact allowlist,
guard self-test, pre-load input hashes, 600-file checkpoint inventory, first
scientific-load timestamp, runner source and compile audit, blocked-access log,
and explicit no-run sentinels. It contains no D3 checkpoint or trajectory and
must never be interpreted as a partially completed scientific fit.

Future readiness work must validate every import and temporary-file behavior
under the same no-delete/no-rename policy before scientific tensor loading.
The forward evaluator also requires an isolated non-Atlas-path metadata
artifact before novel D3 outputs can receive an authoritative forward gate.

# Fixed-Feature Decoder Audit

## Question

Thayer-FF stopped before producing a scientific result because its
repository-integrity gate could not prove that a recursive checkpoint search
avoided prohibited paths. Thayer-RI rebuilt the diagnostic under exact-path
access and independently validated the scientific implementation before
retrying the fixed-feature question.

## Frozen one-scene ladder

All executed conditions used the exact ambiguous training scene, exact P0
target set, unchanged hard assignment, unchanged scientific evaluator, and
matched mapping-specific endpoint states. Every executed condition ran exactly
5,000 MPS optimizer steps without early success stopping.

| Gate | ReLU | Square | Absolute |
| --- | --- | --- | --- |
| D0 | Fail | Pass | Fail |
| D1 | Not run | Pass | Not run |
| D2 | Not run | Fail | Not run |
| D3 | Not run | Not run | Not run |

Square D0 reached own, alternate, and both-mode coverage of 100%. Square D1
did the same, confirming that each restored rank-six final head has sufficient
output span when its penultimate tensor is free. Square D2 used only 204
trainable head parameters, reduced direct target loss by about 28.5%, but
retained zero own, alternate, and both-mode coverage. Its two augmented
feature-design matrices each had rank 17.

## Interpretation

The primary category is **FROZEN-FEATURE CONDITIONING BARRIER**, with first
reachability failure at D2. This is not evidence that a larger decoder is
needed. D3 and tangent diagnostics were blocked by the preregistered progression
rule, so decoder capacity and decoder optimization remain unresolved.

The exact next experiment is a new preregistered one-scene square-mapping
full-L0 fixed-feature diagnostic that retains D2 as the failed control but
prospectively authorizes D3 after the demonstrated D1 pass. It must reuse the
same cache, endpoint, loss, assignment, evaluation, and data restrictions; it
must not open broader data or increase capacity.

## Thayer-D3 prerequisite failure

The prospectively authorized Thayer-D3 campaign reproduced D0, D1, D2, the
cached features, target hashes, and exact square initial state, then stopped
before optimizer construction. The successful D1 artifact persisted output
tensors and metrics but not the two optimized penultimate tensors required for
the frozen feature-endpoint comparison. No D3 or tangent result exists.

This does not alter the frozen-feature conditioning diagnosis or supply a
decoder-capacity claim. The one next experiment is a separate square-only D1
endpoint-persistence replay that saves both penultimate tensors and verifies
their frozen-head outputs. Broader data and added capacity remain closed.

## Thayer-D1R endpoint resolution

Thayer-D1R replayed the exact square D1 condition. All 54 trajectory hashes,
the final objective, raw/mapped/physical outputs, assignment, and 100% own,
alternate, and both-mode coverage reproduced exactly. Four explicitly named
prompt/expert penultimate tensors are now persisted with frozen-head,
fresh-process, and batch/serialization replay proofs.

This resolves the D1 artifact prerequisite but is not a D3 result. A separate
square-only one-scene D3 campaign is now authorized under a new preregistration;
eight-scene fitting and a capacity ladder remain unauthorized.

## Thayer-D3R execution-readiness stop

The complete D1R reference passed the retry's metadata and hash preflight, but
the guarded runtime stopped before optimizer construction after an import
dependency attempted prohibited deletion. This is not a scientific D3 result
and does not change the frozen-feature conditioning diagnosis. Decoder
optimization and capacity remain unresolved.

## Thayer-D3A contract-completeness stop

The authoritative D3A bootstrap matched every frozen runtime and scientific
container hash but found no isolated scientific sky vector or plausibility
thresholds for the forward gate. It stopped before tensor loading, model
construction, or the one-step trace. Decoder reachability, optimization, and
capacity therefore remain unresolved; the D0-D2 evidence is unchanged.

## Thayer-D3S bundle-completeness stop

The new bundle hash and 180-entry count matched, but the registry does not
define the required expert-activity/death gate or complete stop/outcome
protocol. D3S stopped before cached-feature loading and made no new decoder
reachability observation. D0-D2 and D1 endpoint evidence remain unchanged.

## Thayer-D3I observation

Eight exact allowlisted containers were deserialized, but a dtype-token
comparison stopped execution before feature consumption by a decoder. No
model, optimizer, forward, gradient, update, or alignment result exists. The
frozen outcome is implementation/contract failure, not decoder failure.

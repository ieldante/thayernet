# D3 scientific worker contract

The v4 scientific worker requires a bridge path and SHA-256, a fresh output
root, a strict runtime root, and one of two modes: synthetic integration
preflight or authoritative scientific D3. It cannot accept bundle v2
independently.

Before scientific values, the worker validates the bridge, bundle v3,
v3-resolved bundle v2, capsule v1, runtime readiness, 180
architecture/scientific requirements, and 16 executable policies. Policy
thresholds and outcome logic remain in the frozen policy engine; model and
optimizer definitions remain in bundle v2.

Authoritative mode permits only the eight contract-referenced containers used by
the attempted run. Matplotlib is absent from the scientific import graph,
bytecode generation is suppressed, MPS fallback is disabled, and postprocessing
has a separate manifest-only interpreter.

In Thayer-D3I, all eight container hashes matched. The worker stopped while
validating the first D1 endpoint member because the contract token `<f4` and
NumPy display token `float32` were compared literally. Two strict-guard `.pyc`
cache reads were also blocked. No model, optimizer, decoder forward, gradient,
or D3 step followed.

## V4.1 closure

The v4.1 adapter preimported and prewarmed the proven serialization modules,
kept strict permissions unchanged, and passed the previous `<f4`/`float32`
mismatch through canonical comparison. The worker then stopped in added member
inventory before model construction because a higher-rank tensor was passed to
a CHW-only hash. No patch or retry was permitted after source freeze.
